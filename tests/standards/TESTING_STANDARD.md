# MyFinancialAgent 测试总规范

最后更新：2026-03-17

## 1. 文档定位

本文档是项目测试相关的统一总规范，合并了原先分散的：

- 开发 / 测试 / 发布 SOP
- 测试环境说明
- 测试数据与依赖策略
- 测试环境重置与基线方案
- 发布前检查清单

具体测试用例不放在这里：

- 冒烟测试用例：`tests/smoke/`
- 集成测试用例：`tests/integration/`

## 2. 基本原则

- 日常开发默认先在本地验证。
- 阶段联调默认落到测试环境，不直接污染生产资源。
- 非 `master` 分支只允许部署到测试资源。
- `master` 只部署生产资源。
- 文档、配置、测试口径和代码改动必须同步。

## 3. 环境与资源隔离

当前测试相关资源约束：

- 测试 Worker 配置：`tests/testdata/config/wrangler.test.toml`
- 测试环境必须与生产环境隔离：
  - 测试 D1 与生产 D1 分离
  - 测试 Worker 与生产 Worker 分离
  - 测试前端入口与生产前端入口分离

环境关系：

- 本地环境：快速开发验证
- 测试环境：联调、集成测试、阶段验收
- 生产环境：正式服务

## 4. 测试数据与第三方依赖

默认策略：

- 能走真实链路就优先走真实链路
- 辅助测试数据只在真实链路无法稳定覆盖时使用
- 辅助测试数据优先补：
  - `stock_raw`
  - `news_raw_data`
- 当前复盘归档链路新增两类结果层校验对象：
  - `daily_news_ai_analysis.source_news_ids`
  - `daily_review_archive_news`
- 测试环境的集成测试如果涉及历史复盘、多日期列表、跨天状态、历史抽屉或历史页面行为，必须补充足够的历史辅助数据支撑验证，不能只保留当天真实链路数据
- 清库后如果需要执行这类集成测试，默认流程应为：
  - 先重建当天真实链路基线
  - 再补充历史辅助数据
  - 最后执行历史维度相关测试

当前辅助资源统一放在：

- `tests/testdata/`

第三方接口口径：

- 新闻源、行情源、LLM、测试 Worker/API 默认都按真实接口理解
- 第三方异常需要单独归因，不能直接等于项目逻辑失败
- 新闻链路按“允许真实增量、禁止重复污染”解释，不按静态输入强幂等理解
- 已复盘详情页如果切到归档模式，默认应优先读取 `daily_review_archive_news`，而不是继续依赖 live `news_raw_data`

## 5. 测试环境重置与基线

测试环境允许两类操作：

- 完整重置：清空业务表，保留 migration 状态
- 基线重建：从真实链路或辅助测试数据重新建立可验证状态

推荐基线顺序：

1. 真实链路基线
2. 辅助测试数据基线
3. 最小状态基线

如果本轮目标是验证端到端协同，优先使用真实链路基线。

## 6. 开发、测试、发布流程

开始开发前，默认先看：

- 全局记忆
- 项目记忆
- `tests/README.md`
- 当前要执行的测试用例说明

提交前：

- 默认先执行冒烟测试
- 冒烟结果要明确：
  - 执行环境
  - 执行范围
  - 通过 / 失败 / 未执行
  - 失败归因
  - 是否阻断提交

阶段联调时：

- 按集成测试用例执行
- 明确测试环境资源与生产环境隔离
- 记录真实第三方接口参与情况

发布前：

- 必须看冒烟结果
- 必须看集成测试结果
- 必须确认资源隔离
- 必须确认剩余风险已解释

## 7. 发布门槛

发布前至少确认：

- 测试 / 生产部署目标明确
- Worker 和 D1 绑定正确
- 当前环境标识正确
- `/api/health` 可用于环境识别
- 没有测试 token、测试入口或高风险调试逻辑误入生产
- 本轮所需测试已执行
- 第三方依赖风险已说明
- 状态机关键路径已验证：
  - `initialized`
  - `draft`
  - `reviewed`

最终发布结论只能是：

- 可发布
- 有条件可发布
- 不可发布

## 8. 目录分工

- `tests/standards/`
  放统一测试规范
- `tests/smoke/`
  放冒烟测试用例与 UI 测试脚本
- `tests/integration/`
  放集成测试用例与完整测试矩阵
- `tests/testdata/`
  放辅助测试数据
