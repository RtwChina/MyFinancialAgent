标的管理 显示名称：前端展示用，显示在价格卡片、新闻列表、复盘页面。如"标普500"、"英伟达"。与代码区分：代码是系统标识，显示名称是给人看的。

代码：系统唯一标识，用于数据库关联、价格采集、新闻打标。箭头后为 Yahoo 代码（调用价格API用）。前端也作为标签显示，所有页面的标的名（价格卡片、新闻列表、复盘页面）。

类型：仅用于前端分组展示（大盘 / 板块 / 个股）。新闻的最终 type 由 Stage 3 LLM 判断，不由标的类型决定。

新闻匹配别名：新闻正文中匹配到别名时，该标的自动加入新闻的 related_symbols（如出现"英伟达"或"NVIDIA"→ 关联 NVDA），Stage 3 LLM 也会收到这个关联。别名匹配是 rule_score 中权重最高的加分项（权重 3.5），在 Stage 2 与 Embedding 相似度叠加影响保留结果。

与关键词的核心区别：别名回答"这条新闻提到了哪个标的"并产生标的绑定；关键词只影响评分，不产生绑定。两套机制独立叠加。

关键词管理 macro（宏观）：权重 2.5，加分。命中美联储、利率、地缘、战争等宏观关键词，提升 rule_score，在 Stage 2 与 Embedding 相似度叠加，影响新闻最终是否保留。

market（市场）：权重 1.7，加分。命中财报、芯片、AI、并购等市场话题关键词，提升 rule_score。

noise（噪音）：权重 2.5，减分。命中分析师评级、目标价、传闻等噪音词，降低 rule_score，在 Stage 2 压低综合评分。

symbol_context（标的事件）：权重 1.2，加分。配合标的别名匹配使用，命中财报、监管、收购等事件词进一步提升 rule_score。

与标的别名的区别：关键词回答"这条新闻属于什么话题"，标的别名回答"这条新闻提到了哪个标的"。标的别名的权重（3.5）高于关键词，两套机制独立叠加。

基础词（系统预置）只能启用/禁用；自定义词可以删除。
