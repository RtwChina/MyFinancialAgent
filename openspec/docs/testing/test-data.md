# 测试数据与依赖策略文档

## 数据策略
测试数据必须可复现，不允许临时手造且不可复现。

### 可选数据策略
- 固定种子数据
- 脱敏真实样本
- 场景合成数据
- 历史回放数据

## 第三方接口分类

| 分类 | 定义 | 策略 |
|------|------|------|
| A 类 | 本项目可控接口（测试 Worker、自有后端） | 走真实测试环境服务 |
| B 类 | 第三方不可控接口（新闻源、行情源、LLM、外部平台） | mock / 桩服务 / 录制回放 |

## B 类接口说明要求
必须在文档中明确：
- 采用策略（mock / 桩服务 / 沙箱 / 白名单账号）
- 频率限制与成本
- 超时、重试、降级方案
- 第三方异常单独归因，不背锅给项目

## 辅助测试数据规则
辅助数据是补充手段，不是默认方案。使用条件：
- 真实链路无法稳定覆盖目标场景
- 无法稳定构造特定边界条件
- 无法稳定复现某个异常状态

使用原则：
- 优先补未加工的源头数据，不补中间结果
- 尽量保持接近真实原始输入，最小必要构造

## 清理与重置
- 测试数据只写测试库
- 测试后必须可清理或复位
- 覆盖场景：正常、边界、重复、空值、异常状态

## 每次集成测试必须说明
1. 数据来源与生成方式
2. 是否可重复
3. 如何清理或重置
4. 是否污染后续测试
5. 如何与生产数据隔离
