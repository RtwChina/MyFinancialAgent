## Why

当前“个股与资产操作计划”按 `美股 / 大A` 分组，但用户真实决策是围绕股票/基金账户进行：不同账户有不同资金规模、可用资金、币种、交易范围和调仓约束。继续用地域或市场分组会把“标的属于哪个市场”和“资金从哪个账户执行”混在一起，难以支持账户级仓位管理和资金计划。

## What Changes

- 新增股票/资产账户管理能力，用于维护账户名称、机构、账户类型、币种、总资产、可用资金、排序、启用状态和备注。
- 默认支持用户当前 3 个账户：
  - `老虎-美股`
  - `东方财富-国内`
  - `天天基金-国内`
- 将每日复盘“个股与资产操作计划”从按 `marketType` 分组调整为按账户分组。
- 每条操作计划绑定一个账户；账户成为操作计划的主要组织维度。
- 保留标的市场属性，用于价格源、新闻源、标的展示和兼容旧数据，但不再用市场作为操作计划的主要分组维度。
- 账户管理页展示和编辑账户资金信息：
  - 总资产：账户整体规模，包括持仓市值和现金。
  - 可用资金：当前可立刻用于买入、申购或调仓的现金/余额，可能扣除冻结资金或未成交委托。
- 操作计划页面在账户标题处展示账户资金摘要，帮助判断计划是否超出可用资金或账户规模。
- 兼容历史操作计划：旧计划若没有账户信息，应进入可识别的默认账户或“未分配账户”，避免历史数据丢失。

不做项：

- 不接入券商或基金平台自动同步资金。
- 不做真实交易下单。
- 不做复杂持仓明细、成交记录、盈亏归因和资金流水。
- 不改变价格源、新闻源和市场路由的既有职责。
- 不把账户资金做成逐日自动快照；第一版只维护当前账户资金状态，并在操作计划中引用当前值。

## Capabilities

### New Capabilities

- `investment-accounts`: 管理股票/基金账户基础信息、资金字段、启用状态和排序。
- `account-based-action-plans`: 每日复盘操作计划按账户组织、保存和展示，并支持历史市场分组数据兼容。

### Modified Capabilities

- None.

## Impact

- 数据库：
  - 新增账户表。
  - `daily_review_action_plans` 需要新增账户关联字段。
  - 操作计划唯一性需要从 `archive_date + symbol` 演进为 `archive_date + account_id + symbol`。
- Worker API：
  - bootstrap 需要返回账户列表和账户化操作计划。
  - 保存复盘时需要校验并保存操作计划的账户归属。
  - 需要提供账户管理的读写接口。
- 前端：
  - 新增账户管理页或账户管理入口。
  - 复盘抽屉“操作计划”步骤改为账户分组视图。
  - 操作计划编辑器需要选择账户，并根据账户过滤或提示可选标的范围。
- 本地 Python/测试：
  - 测试 schema、迁移脚本和结构化计划读写工具需要同步账户字段。
  - smoke/integration 测试需要覆盖账户分组、账户保存、旧数据兼容和重复标的跨账户场景。
- 兼容性：
  - 旧 `market_type` 字段暂不删除。
  - 旧数据没有账户时必须可显示、可保存、可迁移。
  - `asset_plan` Markdown 摘要仍需生成，摘要中应体现账户分组。
