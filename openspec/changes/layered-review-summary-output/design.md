## Overview

这个 change 同时处理：

1. **上游内容职责重分配**
2. **下游复盘页展示语法统一**

目标不是增加更多内容，而是让现有内容更清楚、更少重复、更符合用户的阅读习惯。

## Current Problems

### 问题 1：三桶输出职责重叠

当前日期级 summary 使用三桶并行：

```text
大盘桶
板块桶
个股桶
```

但三个桶当前都会产出：

- `daily_major_events`
- `sector_impact_map`
- `linkage_logic_chain`

这导致它们容易围绕同一条主线重复表述，例如：

- 大盘桶讲“地缘风险缓和 -> 风险偏好回升”
- 板块桶也讲“地缘风险缓和 -> 科技板块修复”
- 个股桶再讲“地缘风险缓和 -> AI链估值修复”

它们都对，但最终拼装后会显得重复。

### 问题 2：前端层级识别不够直观

目前页面上的内容常常是一句自然语言，用户很难一眼判断：

- 这是大盘层内容
- 这是板块层内容
- 还是个股层内容

### 问题 3：区块展示语法不够分层

当前三个区块视觉语法过于接近，难以建立阅读节奏。

用户明确希望：

- `每日新闻总结` 保留 `# 1.` 风格
- `市场影响` 与 `逻辑链` 使用普通 `1.` 风格
- 三个区块中的每一条都带 `[大盘] / [板块] / [个股]`

## Decisions

### 决策 1：三桶职责重分配

#### 大盘桶
主责：
- 提炼当天宏观、指数、流动性、利率、风险偏好主线
- 输出当天最重要的大盘级结论

次责：
- 少量大盘级逻辑链

不应展开：
- 板块细节
- 个股催化

#### 板块桶
主责：
- 提炼行业、主题、产业链、ETF、轮动结构
- 输出市场影响中的板块内容

次责：
- 少量板块级逻辑链

不应展开：
- 宏观总主线的重复解释
- 个股级长篇输出

#### 个股桶
主责：
- 提炼核心标的催化
- 输出个股层逻辑链

次责：
- 少量个股影响

不应展开：
- 整体宏观主线
- 泛行业总结

### 决策 2：三类层级标签必须显式化

`daily_major_events`、`sector_impact_map`、`linkage_logic_chain` 三个区块中的每一条都必须显式带层级标签：

- `[大盘]`
- `[板块]`
- `[个股]`

这样前端不需要猜测内容层级。

### 决策 3：三个区块使用不同编号语法

#### 每日新闻总结
使用：

```text
# 1. [大盘] ...
# 2. [板块] ...
# 3. [个股] ...
```

#### 市场影响
使用：

```text
1. [大盘] ...
2. [板块] ...
3. [个股] ...
```

#### 逻辑链
使用：

```text
1. [大盘] ...
2. [板块] ...
3. [个股] ...
```

## Prompt Responsibility Shape

### 大盘桶 Prompt

- 重点回答：今天市场主线是什么
- 重点输出：`daily_major_events`
- 可输出少量 `[大盘]` 逻辑链
- 不要长篇展开板块和个股

### 板块桶 Prompt

- 重点回答：哪些板块受影响、方向是什么、为什么
- 重点输出：`sector_impact_map`
- 可输出少量 `[板块]` 逻辑链
- 不要重复宏观主线

### 个股桶 Prompt

- 重点回答：哪些标的值得看、催化是什么、逻辑链是什么
- 重点输出：`linkage_logic_chain`
- 可输出少量 `[个股]` 影响
- 不要复述整体市场主线

## Assembly Strategy

保留三桶并行 + 程序拼装，不增加新的总汇总 LLM。

最终拼装时：

- `daily_major_events`：以大盘桶为主，板块/个股只少量补充
- `sector_impact_map`：以板块桶为主，大盘/个股少量补充
- `linkage_logic_chain`：按大盘/板块/个股各取少量，形成层次

## Frontend Rendering Strategy

前端不再把三个区块都渲染成相同的列表语法。

而是：

- `每日新闻总结`：渲染成 `# 1. [标签] 文本`
- `市场影响`：渲染成 `1. [标签] 文本`
- `逻辑链`：渲染成 `1. [标签] 文本`

## Risks

### 风险 1：上游输出仍然重复

如果 prompt 职责边界不够清楚，三桶仍会重复。

缓解：
- 在 prompt 中显式声明“该说什么 / 不该说什么”

### 风险 2：层级标签不稳定

如果模型没有稳定输出 `[大盘] / [板块] / [个股]`，前端仍需 fallback。

缓解：
- 将标签要求写成硬约束
- 前端保留兜底解析，但目标是减少 fallback 使用率
