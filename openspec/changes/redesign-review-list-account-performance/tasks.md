- [x] 定义复盘列表返回字段：`当日主线` 和 `账户表现`。
- [x] 新增复盘日账户快照数据契约：一个复盘日一条快照，内部用结构化数据保存多个账户。

- [x] 支持用户调整总资产、可用资金、净入金/出金、盈亏和备注。
- [x] 完成复盘时按 `archive_date` 保存或更新一条结构化账户快照。
- [x] 保存快照后同步更新账户管理页的当前总资产和可用资金。

## 1.2 临时历史回填任务

- [x] 单独实现一次性临时任务，处理 `2026-05-08` 以及之后的复盘日。

- [x] 为每个复盘日按 `archive_date` 生成或更新一条结构化账户快照。
- [x] 能计算真实盈亏的账户写入 `dailyPnl` 和 `dailyPnlPercent`。
- [x] 暂时无法计算真实盈亏的账户保留可确认字段，盈亏字段置空。
- [x] 回填任务必须可重复执行，不生成重复快照。
- [x] 回填完成后抽查 `2026-05-08`、最近一个已复盘日、以及一个缺失数据日期。

- [x] 将账户化操作计划映射到账户名称。

- [x] 生成账户估算影响时，关联复盘日标的涨跌幅。
- [x] 将当前仓位范围解析成中位数权重。
- [x] 判断同一账户内的新增标的；新增标的首日不纳入复盘日 K 线估算。

- [x] 列表只展示账户估算影响区间档位，不展示精确估算百分比。
- [x] 历史复盘只有旧 `market_type` 或文本摘要时，不自动猜测账户盈亏。
