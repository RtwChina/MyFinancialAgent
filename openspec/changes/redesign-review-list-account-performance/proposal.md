# 重构复盘列表为账户表现视图

## 背景

当前全量复盘列表的信息密度不太对：

- 每一行都重复展示北京时间交易窗口，扫列表时很占空间。
- `操作计划` 列现在主要是标的拼接，例如 `MSFT | NVDA | ...`，不能直接回答“这一天我的几个账户大概表现如何”。
- `状态` 列太宽，已完成的行也没有必要再展示一个状态标签。
- 用户更常用的是进入编辑，尤其是直接继续调整操作计划，所以 `编辑` 应该比 `查看` 更突出。

列表页应该更像一个交易台工作区：先看复盘日，再看账户表现，再看当日主线，最后快速进入编辑。

## 变更内容

- 行内不再展示 `北京时间 05/15 21:30 - 05/16 04:00` 这类文字。
- 中国时间和美国复盘日的关系，移动到 `复盘日` 表头的提示里。
- 移除独立的 `状态` 列。
- 未复盘通过日期单元格表达：日期置灰，日期下方显示 `未复盘`。
- 已完成行不显示 `已复盘`、`已完成` 这类状态文字。
- 用 `账户估算影响` 或复盘后确认结果，替代原来的操作计划标的拼接。
- 复盘完成时为每个账户记录该复盘日的账户快照，用于后续列表展示真实账户复盘结果。
- 账户名保持中性展示，不根据涨跌上色；颜色只用于数值、条形、热力背景或方向提示。
- `查看` 保留为次要操作，`编辑` 作为主要操作。
- 点击 `编辑` 默认进入该复盘日的操作计划编辑上下文。
- 以 `tests/review-list-redesign-demos.html` 中的 A 系列“交易台表格”方向作为视觉参考继续演进。

## 不做内容

- 不接入券商、基金平台或银行接口。
- 不做交易下单或委托管理。
- 不改变复盘编辑抽屉的主体流程，只调整列表进入编辑的入口语义。
- 不给已完成行增加额外装饰性状态徽标。

## 影响范围

- 复盘列表 API 需要提供更明确的 `当日主线` 和 `账户表现` 摘要数据。
- 前端列表列结构需要从 `操作计划 / 状态` 调整为 `账户表现 / 当日主线 / 操作`。
- 需要补充针对右侧内容被隐藏、按钮错位、账户表现缺失等问题的布局回归测试。
