## Overview

这个 change 只处理一件事：**收紧 Stage 3 的 LLM 打星标准**。

它不改：

- 规则层 `_score_to_stars()`
- 三桶日总结配额逻辑
- canonical news type

这样可以把“为什么高星膨胀”单独隔离出来观察。如果效果明显，再决定是否还需要继续下调规则层映射阈值。

## Current State

当前最终星级的形成路径是：

```text
规则层初筛
  -> 规则分数映射初始星级
      -> Stage 3 LLM 重新判断 keep/type/importance_stars
          -> 最终写入 news_raw_data
```

因此，虽然规则层也在参与，但最终用于 summary 和复盘页的 `importance_stars` 主要由 Stage 3 LLM 决定。

当前 prompt 的问题有两个：

1. 4 星定义偏宽  
`4星（重要，值得重点复盘）` 这个描述容易把大量“重要但非核心”的新闻抬成 4 星。

2. 分布约束偏宽  
当前 prompt 允许：
- 5 星最多 20%
- 4-5 星合计最多 40%

这对于批次新闻筛选来说过于宽松，尤其会导致大盘类新闻在高波动窗口里集体抬升。

## Decisions

### 决策 1：重写 5 星 / 4 星 / 3 星语义

新的语义：

- 5 星：改写当日交易主线的极少数事件
- 4 星：值得写入当天复盘重点段落的少数事件
- 3 星：有信息增量、值得纳入复盘，但不属于最核心输入层

这条决策的本质是把 4 星和 5 星从“比较重要”改成“极其克制的少数高优先级输入”。

### 决策 2：将高星分布约束改为硬约束

批次级别的硬约束调整为：

- 5 星：最多本批新闻的 `5%`
- 4 星：最多本批新闻的 `10%`
- 4-5 星合计：最多本批新闻的 `20%`

补充说明：

- 这是硬约束，不是建议值
- 如果模型认为需要超过该比例，必须先重新校准评分标准，而不是直接给更多高星

### 决策 3：增加“禁止高星”的负面规则

以下内容通常不得高于 3 星：

- 同一主线事件的重复快讯、后续跟进报道
- 没有新增事实、只有评论或媒体解读
- 仅描述盘中涨跌，没有新增原因
- 分析师评级、目标价、媒体汇编、摘要类内容

### 决策 4：增加“同主线降星”原则

如果多条新闻属于同一主线事件：

- 只有最关键、信息最完整、最具决定性的少数新闻可以给 4-5 星
- 其余同主题新闻原则上降为 2-3 星

这条规则尤其针对当前 `index` 类型高星膨胀的问题。

## Prompt Shape

Stage 3 prompt 中 `importance_stars` 部分建议收敛为：

```text
5星：极少数，改写当日交易主线
4星：少数，值得进入当天复盘重点输入层
3星：主体，有信息增量但不属最核心输入
2星：背景补充
1星：弱相关
0星：噪音
```

同时附加：

```text
本批新闻的星级分布必须严格克制：
- 5星：最多本批新闻的 5%
- 4星：最多本批新闻的 10%
- 4-5星合计：最多本批新闻的 20%
如果你认为更多新闻应得到 4-5 星，说明你的评分标准过宽，必须重新校准。
```

## Verification

这个 change 的效果应通过以下方式观察：

1. 单批次日志抽查  
观察每个 batch 的 4 星和 5 星数量是否明显下降。

2. 窗口级统计  
对一个真实复盘窗口统计：
- 5 星总数
- 4 星总数
- 4-5 星合计
- `index / sector / stock` 的高星分布

3. 日总结候选池变化  
观察 `stars>=4` 的 summary 候选数量是否明显下降，尤其是 `index` 类型是否回到更合理范围。

## Risks

### 风险 1：模型过度收缩
如果约束过于严格，模型可能把很多本应是 4 星的新闻压回 3 星。

缓解：
- 先只改 prompt，不改规则层阈值
- 用真实窗口数据观察一轮

### 风险 2：机械凑分布
模型可能为了满足比例约束而机械打分。

缓解：
- 保留清晰的语义定义
- 结合负面规则和锚定示例，而不是只给比例
