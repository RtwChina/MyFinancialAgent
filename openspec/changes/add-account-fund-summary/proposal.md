## Why

用户现在按账户维护操作计划和资金规模，但缺少一个能把账户资金、标的金额、跨账户占比集中看清楚的汇总视图。随着 `老虎-美股`、`东方财富-国内`、`天天基金-国内` 三个账户都开始维护总资产、仓位区间和标的金额，继续只在单个账户卡片里查看会难以判断整体资金分布。

## What Changes

- 新增“资金分布汇总 / 统计汇总”能力，用于按币种汇总账户资金和标的资金分布。
- 汇总页按币种分组展示：
  - USD 美元账户单独汇总。
  - CNY 人民币账户单独汇总。
  - 第一版不做美元和人民币之间的汇率折算。
- 每个币种分组展示：
  - 总资产。
  - 估算持仓金额。
  - 未分配金额。
  - 账户覆盖情况。
- 每个币种分组内展示标的矩阵：
  - 行为标的。
  - 列为账户。
  - 单元格展示该标的在该账户中的金额和占账户总资产比例。
  - `总金额` 汇总同币种所有账户下同一标的金额。
- 标的金额计算规则：
  - 优先使用结构化操作计划里的手填 `position_amount`。
  - 若未填写具体金额，则按账户总资产 × 当前仓位区间中位数估算。
  - 估算金额后显示 `*`，tooltip 说明“按仓位区间中位数估算”，并展示区间到中位数的映射。
- 统计范围第一版以当前账户活态操作计划为主，即 `account_live_action_plans`。
- UI 风格参考 `tests/summary-demo.html` 当前确认方向：资金分布汇总、币种分组、账户覆盖侧栏、标的矩阵、低干扰 `*` 估算提示。

不做项：

- 不接入券商或基金平台自动同步持仓。
- 不做真实收益、成本、成交、盈亏归因。
- 不做 USD/CNY 汇率折算和跨币种合计。
- 不把自由文本里的金额解析为结构化金额。
- 不在第一版实现历史复盘日资金分布对比；历史视图可后续基于 `daily_review_action_plans` 扩展。

## Capabilities

### New Capabilities

- `account-fund-summary`: 按账户和币种汇总当前活态操作计划中的标的资金分布，并对缺失金额的标的按仓位区间中位数估算。

### Modified Capabilities

- None.

## Impact

- 数据库：
  - 复用 `investment_accounts.total_assets`、`investment_accounts.currency`、`account_live_action_plans.position_amount`、`account_live_action_plans.current_position`。
  - 不新增必须表；如实现需要缓存，可后续评估，但第一版应实时计算。
- Worker API：
  - 新增资金分布汇总读取接口，或在现有账户/活态计划接口基础上组合出汇总数据。
  - 需要统一金额单位：账户总资产和标的金额存储为原始货币单位，前端以“万”为输入/展示单位。
- 前端：
  - 新增统计汇总入口或账户页内的汇总视图。
  - 新增币种分组、账户覆盖、标的矩阵和估算 tooltip。
  - 保持桌面端无页面级横向溢出；账户列较多时仅矩阵区域横向滚动。
- 测试：
  - 覆盖手填金额优先、仓位区间估算、USD/CNY 分组、同一标的跨账户汇总、`*` tooltip、账户总资产万元单位。
- 兼容性：
  - `position_amount` 为空的历史或活态计划不丢弃，按仓位估算。
  - `current_position` 为 `0%` 或准备开仓类动作时金额为 0 或不展示持仓金额。
  - 缺少账户或账户总资产异常的计划应明确降级显示，而不是参与错误汇总。
